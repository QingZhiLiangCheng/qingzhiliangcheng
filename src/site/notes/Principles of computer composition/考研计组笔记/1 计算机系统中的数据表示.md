### 数据表示与数据编码的有关概念
计算机世界，Everything is bits, 每一个bit都是0或者1，但是对于我们来说，更习惯用十进制，为什么计算机用二进制存储数据？
处于电子实现的考虑，双稳态原件非常容易制造
把低电压区间认为是0，高电压区间认为是1，就算电路中存在噪音或者其他干扰，只要不超过设置的阈值，我们就能清晰的得到0和1 -- 稳定
![Pasted image 20260506203327.png\|500](/img/user/accessory/Pasted%20image%2020260506203327.png)

在计算机中所有的数据都是二进制！ 但是如果我们直接用二进制就变得特别冗长 用十进制又难以转换为01 所以计算机的设计者最终选择了使用十六进制 十六进制的好处在于每四位二进制数可以很容易的转换为一个十六进制数，一个十六进制数可以很简单的写成四个二进制数

计算机的基本存储单位是Byte 字节 1字节=8bit 我认为原因是当时设计ASCII码有127个字符，可以刚好由7个二进制位表示，而8个二进制位能包含所有的ASCII更好实现且能够用最高位做奇偶校验和扩展，所以规定一个基本存储单位为8字节

word, 字：字长，比如64位机和32位机 是计算机CPU一次性处理、传输或存储的信息量的最大长度

在内存地址空间中，每个地址对应的是一个字节，而地址是无符号数，意味着对于64位机器 地址范围就是从64全0到64全1

计算机中需要表示、处理的数据: 
- 指令
- 数值数据：正数、负数，整数、小数、实数
- 文字、字符，声音、图像、视频

这些东西都是用二进制存储的，所以有个概念叫做位模式，所谓的位模式，就是一位一位的0101代码，这段二进制可能是一个数，也可能是一个字母，也可能是一个图像，比如同一段二进制数所表示的无符号数和有符号数就可能是两个数 这是相同位模式的不同解释方式

数据表示 vs. 数据结构
- 数据表示:指的是能由机器硬件直接识别和引用的数据类型。由硬件实现的数据类型
- 数据结构:面向计算机系统软件、面向应用领域所需处理的数据类型。由软件实现的数据类型。
	- 目标:最大限度满足应用要求、最简化的方法实现。
	- 实现:通过数据表示和软件映象相结合方法实现。


编码:采用少量的基本符号，选用一定的组合原则，以表示大量复杂多样的信息。
计算机内部的数据若有确定的值，即在数轴上能找到其对应的点，则称为数值数据。
数值数据是表示数量多少和数值大小的数据。

### 原码
核心：符号位+绝对值
![Pasted image 20260507150543.png\|500](/img/user/accessory/Pasted%20image%2020260507150543.png)
对于原码定点小数和定点整数有严格的定义，我们来分析这个定义
对于定点小数来说，如果是个正数，那一定就表示成$0.000...000$的形式，这个数本身就是他的源码 小数点前面的0表示符号位，0表示正数；如果是个负数，那么他的绝对值是$0.000...000$的形式，+1变成$1.000...000$,第一位1为符号位，1表示负数
对于定点整数来说，要注意表达式的范围，比如第一个范围区间是$0\leq X<2^{n-1}$, 所以对于n位原码来说，只能表示$0\leq X \leq 2^{n-1}-1$这些数
$$2^{n-1}=1\underbrace{000 \cdots 0}_{n-1 \text{ 个 } 0}$$
$$2^{n-1}-1=\underbrace{111 \cdots 1}_{n-1 \text{ 个 } 1}$$
其实这也不难理解，因为第一位必须是符号位，正数为0，剩下的数值位只有n-1位，所以正数所能表示的最大值为$0\underbrace{111 \cdots 1}_{n-1 \text{ 个 } 1}$
对于负数情况也很好理解了

另外要注意的是不管是在定点小数还是定点整数的定义中，0在两段区间中都存在，也就是说0定义了两次，这也很好理解，比如在整数中$0,000 ... 0$这是正0, $1,000 ... 0$这是负0，通常情况下认为是相等的

表示范围
首先要知道原码是对称的，所以我们看正数就好，会好看一点
对于定点小数，最大值就是$0.\underbrace{111 \cdots 1}_{n-1 \text{ 个 } 1}$,用另一种方式写就是$1.\underbrace{000 \cdots 0}_{n-1 \text{ 个 } 0}-0.\underbrace{000 \cdots 1}_{n-1位}$即$1-2^{-(n-1)}$, 那取值范围就是$-(1-2^{-(n-1)}) \leq X \leq 1-2^{-(n-1)}$
定点整数同理, 最大正数为$0\underbrace{111 \cdots 1}_{n-1 \text{ 个 } 1}$,即$1\underbrace{000 \cdots 0}_{n-1}-0\underbrace{000 \cdots 1}_{n-1}$,即$2^{n-1}-1$,那取值范围就是$-(2^{n-1}-1)\leq X \leq2^{n-1}-1$

根据上面的分析，不难发现 字长为n，总共有$2^n$个码点，但对应的真值只有$2^n-1$个
负数的原码的编码值是大于正数的原码的编码值

原码是很容易实现乘法的，同号相乘结果为正，异号相乘结果为负，很好实现，用异或门，$A\oplus B$，然后再算绝对值的乘积就好了
但是原码实现加减运算是比较复杂的，对于异号加法来说，要比较加数和被加数的绝对值，取绝对值较大的数的符号，然后结果的绝对值是绝对值较大的减去绝对值较小的 --麻烦，设计硬件的时候实现加法还需要减法

人们就想能不能把加法运算也转换成减法运算 -> 补码表示法
### 补码
#### 补码Overview
想要在硬件中只设计一个加法器，将加法和减法都转化成加法运算来做，并且加减法运算的使用频度是非常高的，所以我们希望加减法最好能在一个时钟周期内完成
生活中很多有模运算体系的例子，钟表就是其中一种
所谓的有模运算是指$A=B+K\times M$,我们就能表示为$A \mod M=B$ 
![Pasted image 20260507160259.png\|500](/img/user/accessory/Pasted%20image%2020260507160259.png)
在钟表的例子中，假设我们想从8点拨到5点，一种方式是逆时针波动3个格子，对应的是减法，即$8-3=5$
当然由于是有模体系，也能表示为$8-3+12=8+9$,即顺时针播9个格子
$8-3$和$8+9$是等效的，我们可以把$8-3$写成$8+(-3)$的形式，所以我们定义-3的补码是9！-3即加-3的补码9
所以在有模体系中，减去一个数等于加上这个数对模的补数，其中假定M为模，若$a+b=M$,则称a,b互为补数
eg: 在10进制假设只有2位 模为100的体系中，能表示00~99这些数，那计算64-5就是计算64+(5的补数为95)，即64+95=159 mod 100 =59 其实在硬件中，只有2位的话，这样就溢出了高位的1就被挤出去了舍掉了
计算机的2进制本身就是一种有模体系，因为字长是有限的，假设字长为4,1111+1=0000 硬件只有4位，高位被挤出去舍掉了
![Pasted image 20260507161919.png\|300](/img/user/accessory/Pasted%20image%2020260507161919.png)

从另一个层次看，再看10.0101100的真值的时候，末位的100往左都要取反的，所以高位上的0实际上是代表了绝对值上的1，也就是说小数点前面的0实际上代表了真实数字的1，那这个数实际上就不是纯小数了，就没法用定点小数表示了 -- 其实就是移动过程中这个数真值收到影响了 在小数点第一位有权重的 但移出去了
再从另一个角度看，实际上看舍掉这个符号位有没有影响，看的就是新来的这一位和原来的符号位一不一样（这一位能不能当新的符号位） 如果一样，那就可以舍掉符号位，如果不一样，真正的符号位就不能舍掉，就是溢出情况 -> 双符号位！
双符号位补码（变形补码）：左符是真正的符号位，右符是用来判断“溢出”的，在做运算时，比如乘法 加法
- 若运算结果的两个符号位相同，则不发生溢出；
- 如果两个符号位不同，则结果溢出，此时最高位为符号；次高位为溢出的数值

如果溢出了，将CPU中状态寄存器的溢出标志位置位，转入溢出处理程序进行相应的处理

### 移码
#### 移码Overview
计算机中有符号整数都是用补码表示的，而对于一个任意的浮点数，其实前面也提到过一点，对于$1101.0101_2$一般是转换成尾数是纯小数，阶码是纯整数的形式，即$0.11010101_2 \times 2^4$, 这个尾数叫做规格化的尾数
所谓的规格化是化到小数点右边第一位是1，当然对于这个例子也有别的写法比如$0.0011010101_2 \times 2^6$, 但计算机的字长是有限的，我们希望尽可能保留精度
![Pasted image 20260508163757.png\|300](/img/user/accessory/Pasted%20image%2020260508163757.png)
底数2是默认的，不需要存储；计算机中真正需要存储的是，符号位，尾数，阶码；尾数是小数，阶码是整数
浮点数的运算中乘法减法也比较频繁，包括要比较两个浮点数大小方便，所以规定了尾数在计算机中用原码的形式存储
![Pasted image 20260508164633.png\|300](/img/user/accessory/Pasted%20image%2020260508164633.png)
这么存储有一个问题：就是我们会频繁的比较两个数的大小或者说绝对值的大小，而且我们希望对于整数和浮点数用一套电路来实现
整数是怎么比较的？
从高位往低位比

那既然想用同样的电路，那也希望从高位往低位比，但是浮点数我们要先比较阶码 所以干脆把阶码放到前面了

### IEEE754标准
其实前面能感受到 浮点数的格式定义各不相同 这样在移植的时候 就会出现问题 -> 一个统一的浮点数格式标准 -- IEEE754
![Pasted image 20260513103937.png\|500](/img/user/accessory/Pasted%20image%2020260513103937.png)

首先IEEE754标准依然是数值位 阶码 尾数的形式，并且小数点在阶码和尾数中间
用C语言编程的时候，我们经常会定义一个单精度的float变量，单精度的float变量是32位，阶码是8位，只需要记住总长度和阶码的长度，就能推算出尾数是23位
同理，C语言中的双精度double总长度为64位，记住阶码是11位，能够推算出尾数是52位
IEEE754规定基值默认是2 与前面的浮点数一般格式相同
IEEE754规定尾数用原码，这是因为原码是符号位+绝对值的表示方法，在算乘除和比较绝对值大小的时候比较方便，在IEEE754中的标准中，要求的原码是S1.xxx...xxx的形式，有点像十进制科学计数法的形式，小数点左侧有且仅有一个数值位且必须为1，既然必须有这个1，那干脆可以默认隐含不存了，那实际上对于单精度来说就变成了小数点右边的数值存23位，这也就意味着，用23位表示了24位的精度！
阶码：IEEE754规定阶码也是用移码来表示，但是之前讲过的标准移码的偏移量是选的10....0那一个，IEEE754标准所规定的移码偏移量是01....1这一个 -- 这就造成了两个结果，其一是表达式的范围正的方向比负的方向多表示一个（标准移码是+127~-128，而IEEE754阶码这里是+128~-127）；其二是IEEE754的阶码的移码和补码的关系就不是之前将的符号位取反了！
IEEE754所规定的标准，尾数在小数点左侧隐含了一个1，这样就没办法表示机器0了 -- IEEE754中将阶码的最大的编码值全1和最小的编码值全0，不用来表示一个具体的编码值了，而是用它来作为一个特殊标记 -- 全1的编码不用来表示128了，同样作为阶码来讲-127也表示不了 -- IEEE7548位阶码的表示范围实际上是+127~-126
阶码全0的特殊标记代表了浮点数的尾数小数点左侧没有隐藏的1了，即S0.xx...xx 小数点右边还是23位
- 如果小数点右边23位全0，那表示的就是机器0，由于存在符号位，所以应该是+-0
- 小数点右侧23位不是全0，这个时候尾数表示的是非规格化数，阶码值是一个默认的值，是能够正常表示的阶码范围内最小的一个，即-126

阶码全1
- 如果后面的尾数全0的话，表示的是+-∞
- 如果后面的尾数不是0的话，s11111111yy...yy的形式，表示的是NaN -- Not a Number -- 啥叫非数 比如说$\sqrt x, x<0$的情况 

为啥移码的“0”取在了0111..1的位置，这是因为其实有一个运算还是很频繁的:$x\to\frac{1}{x}$
最小的x的阶码可以表示到-126 取倒数 翻上去 阶变成+126不会上溢 因为还能表示+127嘞
那如果很大的阶码，x的阶是+127 一取倒数变成了 -127 那就下溢了啊，但是下溢比上溢是好处理的，下溢可以看做是一个非常接近于0的数，可以返回0啊。但IEEE754做了一个更好的处理，他还能表示S.0xxx...x非规格化的数呢，能返回一个接近于0更精确的值！

### 海明码
计网中讲了
![Pasted image 20260518092822.png\|500](/img/user/accessory/Pasted%20image%2020260518092822.png)
设计一种编码 纠正1位错误 需要多少位？
假设信息码本身m位，我们希望加上r位校验位，也就是说$2^r$能表示出所有的出错可能，而我们希望指明出错的位置进行纠错，总共的出错位置: 信息码出错有m种位置，校验码出错有r种位置，加上没有出错的1中情况，即
$$
2^r\geq (m+r+1)

检错能力：
有一个概念叫做最小海明距离
![Pasted image 20260518102724.png\|400](/img/user/accessory/Pasted%20image%2020260518102724.png)
因为每个信息位至少被两个校验位所校验，所以实际上如果有一位信息位错了，要想变成另一个合法的数据至少要再改变两位校验位：比如6从1变成0, 2和4的位置上也要改变 -> 最小海明距离是3 -> 两个合法编码最少有三位不一样
如果只有一位出错了，实际上现在的这个数是离着原来想传的那个数的海明距离是最近的，没偏离到离着别的合法数据近去，所以计算机会知道你想表示的真正的数，所以能实现一位纠错
如果编码字在传输过程中，如果两位出错，可能就离着另外一个编码字更进了，比如这里的6和4都出错了 就会离着6本来就是0的那个合法字更近，就纠不正确，但是能检错出来
3为出错 就可能直接变成另一个合法字了
