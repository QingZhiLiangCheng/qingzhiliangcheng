### Overview
自治区域之间的路由 -- BGP -- 边界网关协议
之前所讲的RIP和OSPF都是自治区域内的路由
自治区域或者叫自治系统(Autonomous System, AS)? 自治区域的必要性？
之前所学的RIP，OSPF都是在一个平面解决路由的问题

### 层次路由
层次路由  -- 即一个平面的路由 -- 网络中所有路由器的地位一样
也就是说，链路状态算法的话，我的链路状态分组，（在这个平面）所有的状态分组都要知道；距离矢量算法的话，我要把我到所有节点的距离矢量（知道就知道，不知道就无限大）传给我的邻居
这都是在一个平面内解决问题
在整个互联网的范围内，甚至以后IPv6会更多的子网，不管采用链路状态算法还是距离矢量算法都比较困难
除此之外，还有一个管理问题，比如机构的网络希望按照自己的方式管理网络并且希望对外不可见
总结：一个平面解决路由问题：对传输代价和管理来说都是不可行的
-> 互联网的解决方式：把大的路由问题分成两个层面 -- 不同的机构比如聊大构成一个自治区域，自治区域内是一个层面，每个自治区域内可以选择自己的路由选择协议和算法；另外一个层面是自治区域间，每个自治区域实际上可以收敛到数量很少甚至是一道可达信息向外进行发送，所以在自治区域间而言，一个自治区域相当于一个点
如果自治区域内很大可以再次切，就比如上一节介绍的链路状态分组的骨干区域和普通区域的问题
一个自治区域相当于一个点，就把规模降下来了，但是规模仍然是自治区域间路由需要考虑的一个问题
但层次路由就是解决了规模性问题，解决了管理性问题

### 互联网AS间路由：BGP
自治区域之间要统一 -- BGP -- Border Gateway Protocol 边界网关协议 - 是自治区域之间的路由一个事实上的标准 所谓的事实上的标准意思是不是由某一个机构规定出来的，而是由约定俗成的不断的演化所形成的
BGP -- 将互联网各个AS粘在一起的胶水
BGP分成两个部分
- eBGP
- iBGP

每个自治区域是由网关路由器所连接起来的
eBGP是指，我收集我这个自治区域的子网可达信息。怎么收集的？ --  网关也属于自治区域内部，也参与自治区域内部的路由选择算法的运算，也会拿到对应的链路状态分组和距离矢量
iBGP是指，通过TCP的连接将获得的子网可达信息告诉自治区域内部的其他路由器，包括其他的网关路由器
![Pasted image 20260310174240.png|500](/img/user/accessory/Pasted%20image%2020260310174240.png)

BGP允许子网向互联网其他网络通告“我在这里” -- 我有子网前缀是xxx，关于这些子网前缀你都可以通过我来发送，我是你关于这些子网的下一跳
BGP基于距离矢量算法 -- 之前介绍过距离矢量算法有无穷计数和乒乓回路的问题 -- BGP是一种改进的距离矢量算法 -- 关于这些子网，我不仅仅告诉我到他们的代价值，还告诉要穿过的自治区域的序列 -- 所以不仅仅告诉代价，还告诉路径
![Pasted image 20260310174848.png|500](/img/user/accessory/Pasted%20image%2020260310174848.png)
在上面这个图中，虚线是TCP的连接。1c，2a这样的是网关，1c网关既参与到自治区域1内部的计算，还通过eBGP收集自治区域1内部子网可达的信息告诉2a: "2a你不是网关吗，咱也是网关啊，我连了这些子网，有这些子网可达信息我告诉你，关于这些子网前缀，你都可以通过我进行转发" -- 这相当于对子网可达信息的承诺，表示向2a说明，你可以通过我(1c)的ip作为你的下一跳来到达这些子网
2a收到信息之后通过TCP的连接 通过蓝色的虚线IBGP告诉该自治区域内的路由器，包括网关路由器
2c能够告诉3a什么？ 一个是关于AS2的可达信息；一个是转的关于AS1的可达信息，并且转的时候要加码（代价会变）

### BGP基础
两个BGP路由器在开机之后就会建立起TCP连接 -- 半永久TCP连接 -- 在这个连接之上双方交换BGP的iBGP, eBGP的报文，在这些报文当中会包括一些子网可达信息 -- 端口179的半永久TCP连接（交换信息后仍保持连接）
AS3中X上线，3a能够感知到(内部网关协议)，3a向2c通告：你通过我到达X可达，穿过的自治区域路径，我是你关于X的下一跳... 
3a向2c通告：AS3,X -- 只经过了一个自治区域AS3， AS3向AS2承诺，它可以向子网X转发数据报；3a是2c关于X的下一跳
其实AS3就既告诉了路径也告诉了距离
![Pasted image 20260310185552.png|500](/img/user/accessory/Pasted%20image%2020260310185552.png)
通告包括子网前缀和一些属性，属性包括自治区域的路径，下一跳
自治区域之间的路由和自治区域内部的路由不一样，自治区域之间的路由更加强调策略 -- 所谓的策略就是网关B把信息传给网关A，网关A到底是接受这个路由信息还是过滤掉，取决于策略之间的配置；网关A要不要向内部通告是策略；网关A所在自治区域的另一头的网关要不要转，也是策略

### BGP路径通告
![Pasted image 20260310190535.png|500](/img/user/accessory/Pasted%20image%2020260310190535.png)
整个过程是这样的：当AS3中X子网加入之后，由于内部网关协议的存在，3a实际上会感知到X的存在，并且拿到关于X的链路状态分组或者是距离矢量，然后3a会通过半永久TCP连接通过eBGP的方式将AS3自治区域内的各个子网的可达信息告诉2c，比如在图中是由于新增一个子网触发的通告，传达的关于X的路径通告就是AS3,X -- 只经过了一个自治区域AS3， AS3向AS2承诺，它可以向子网X转发数据报；3a是2c关于X的下一跳
2c根据策略决定是接受还是过滤掉；并根据策略决定要不要把这个通告信息通过iBGP的方式传给2a(2a也是网关), 2b, 2d. 
2a一方面综合了AS2的子网的可达信息，一方面接收到了2c传来的关于AS3的可达信息，2a会根据策略，决定向2a通过eBGP的方式通过AS2, AS3, X（路径上加了AS2自己这个自治系统），同时2a也会发送综合的自己的自治区域内子网的可达信息
![Pasted image 20260310191536.png|500](/img/user/accessory/Pasted%20image%2020260310191536.png)
网关路由器可能获取有关一个子网X的多条路径，从多个eBGP会话上AS1网关路由器1c从2a学习到路径：AS2,AS3,X；AS1网关路由器1c从3a处学习到路径AS3,
基于策略，AS1路由器1c选择了路径：AS3,X，而且通过iBGP告诉所有AS1内部的路由器
注意这里并不见得一定就选择上面那条路径，因为它不只有跳数着一个因素，他会对每一种路径进行打分，选择一个最优的

1d作为一个子网内普通的路由器，怎么知道到达其他子网特别是其他自治区域内子网的信息，比如X？
AS1的网关会知道怎么到达X啊，会通过iGBP传给1d，1d会知道想要到子网X转给哪个网关（该网关到达X所在网关的代价最小），然后只需要在自治区域内部通过内部网关算法选择最优路径即可

![Pasted image 20260310194838.png|500](/img/user/accessory/Pasted%20image%2020260310194838.png)
当然不一定X子网就是单独一条 会做聚合

**BGP报文**
- OPEN:打开TCP连接，认证发送方
- UPDATE:通告新路径(或者撤销原路径) 
- KEEPALIVE：在没有更新时保持连接，也用于对OPEN请求确认 
- NOTIFICATION:报告以前消息的错误，也用来关闭连接

![Pasted image 20260313170935.png|550](/img/user/accessory/Pasted%20image%2020260313170935.png)

**BGP路径选择**
路由器可能获得一个网络前缀的多个路径，路由器必须进行路径的选择，路由选择可以基于： 
1. 本地偏好值属性:偏好策略决定 -- 像是自定义的函数 可能经过这个序列打分就高 不想经过某些序列 这可能取决于政治因素 经济因素
2. 最短AS-PATH：AS的跳数 
3. 最近的NEXT-HOP路由器:热土豆路由 -- hhh烫手的土豆 谁离得近给谁 --  如果一个普通路由器通过两个网关到达X子网的打分和AS-PATH相同，就选自治区域内部近的NEXT-HOP
4. 附加的判据：使用BGP标示

如果热土豆策略这个NEXT-HOP还一样，接着选，最后随机，保证最后能选一条路径即可
